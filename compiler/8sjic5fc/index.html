从DFA到正则表达式

我们在之前的NFA一节，引入了一张图。

图中的Tompson's construction，Subset construction和DFA Minimization都在前面的章节中聊过了，还剩下从DFA到RE之间的Kleen's construction。

算法假设DFA的状态编号从 000 到 ∣D∣−1|D|-1∣D∣−1，DFA的接受状态集合为 DAD_ADA​。

我们将符号 RijkR_{ij}^kRijk​ 来表示描述所有从 did_idi​ 到 djd_jdj​的路径上而不经过编号高于dkd_kdk​ 的状态的正则表达式。这里"通过"的意思是，从 did_idi​ 到 djd_jdj​ 之间的路径上，任何状态的编号都小于等于 dkd_kdk​。假设一个DFA具有一个状态转移 d1→d16d_{1} {\rightarrow} d_{1 6}d1​→d16​,R1,162R_{1,16}^2R1,162​ 将不会为空。

Kleen's construction的执行过程大概如下：

  1. 初始化，算法将所有从 did_idi​ 到 djd_jdj​ 的直接路径，记作 Rij−1R_{ij}^{-1}Rij−1​。当 i=ji = ji=j 时，Rij−1R_{ij}^{-1}Rij−1​ 为 ε\varepsilonε。
  2. 进行迭代，每次迭代都会构建更长的路径。每次通过添加通过 dkd_kdk​的路径，利用 Rijk−1R_{ij}^{k-1}Rijk−1​ 来计算 RijkR_{ij}^kRijk​。
    • 从 did_idi​ 到 dkd_kdk​ 的所有路径的集合，这些路径不经过编号高于k-1的状态，与 dkd_kdk​到自身的任意路径相连接，这些路径也不经过编号高于 k−1k-1k−1 的状态，再与从 dkd_kdk​ 到 djd_jdj​ 的所有路径的集合相连接，这些路径同样不经过编号高于k-1的状态.

    • Rijk←Rikk−1 (Rkkk−1)∗Rkjk−1 +Rijk−1R_{i j}^{k} \gets R_{i k}^{k-1} \ ( R_{k k}^{k-1} )^{*} R_{k j}^{k-1} \ +R_{i j}^{k-1} Rijk​←Rikk−1​ (Rkkk−1​)∗Rkjk−1​ +Rijk−1​

  3. 当 kkk 循环终止时，不同的 RijkR_{ij}^kRijk​ 计算了图中所有的路径。
  4. 此时从所有路径中，找到从起始状态 d0d_0d0​ 到接受状态 dj∈DAd_j \in D_Adj​∈DA​ 的路径，这些路径构成了DFA的正则表达式。
